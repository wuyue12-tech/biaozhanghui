# 协作规则：八升九入境仪式策划

本项目当前主线文档是：

- `docs/ceremony/入境仪式成功标准.md`

之后生成、修改或评估任何入境仪式方案时，先以这份文档校准。

## 当前主线

学生共创，团队闯关，家长见证，导师认证。

## 输出要求

每个方案都要包含“成功标准校准”部分，逐项回应：

1. 学生前期共创了什么。
2. 学生现场如何团队参与。
3. 家长在现场看到什么、完成什么。
4. 老师如何组织、判断和认证。
5. 篮球馆空间如何被使用。
6. 活动如何产生热血感。
7. 学生最后带走哪一个具体九年级行动。

## 判断规则

如果方案开始偏向单纯表演、单纯说教、复杂桌游规则或低参与度展示，应回到 `docs/ceremony/入境仪式成功标准.md` 的“方案准入线”重新校准。

## 本轮问题复盘

目前方案暴露出的主要问题：

1. 方案语言先于现场动作。文本里有卡包、路线、挑战、认证等词，但现场谁拿、谁跑、谁写、谁判断、结果如何还不够清楚。
2. 游戏化元素先于教育逻辑。接力、传球、盲盒、路线卡等形式需要服务同一条主线：困境识别、行动生成、支持获得、导师认证。
3. 概念增加了理解成本。例如“信号”应改成学生、家长、老师都能直接理解的“九年级困境”。
4. 每关之间的因果关系需要更硬。上一关的产出必须成为下一关的输入，最后都汇入同一张“九年级行动牌”。
5. 评价标准需要前置。每队怎样算完成、怎样算质量高、谁来判断、判断依据是什么，必须在方案正文里写明。
6. 学生能力水平差异需要被方案吸收。任务应提供卡片、模板、句式、岗位分工，让能力弱的小队也能完成，让能力强的小队能做出质量差异。
7. 现场执行视角需要优先。每个环节都要经得起“老师提前 20 分钟看懂、学生 5 分钟进入状态、篮球馆现场能控住”的检查。

## 后续协作规则

当我对方案表达疑问、不满或指出“经不起推敲”时，先暂停扩写方案，按以下顺序回应：

1. 真正目标是什么。
2. 当前卡点是什么。
3. 哪个分支开始抢占主线。
4. 现在应收束、改名、合并或延后的内容是什么。
5. 下一步最小有效动作是什么。

继续推进方案前，先把主线说清楚：

- 当前真正目标是什么。
- 当前正在推进哪条主线。
- 哪些内容只是表达、道具、视觉、流程或技术分支。

## 全局思考规则

设计活动方案时，先从全局推导，再进入局部机制。

每次提出分队方式、赛道数量、物料摆放、卡片规则、支持点、展示方式或评价方式前，必须先回答：

1. 这个设计服务哪个成功标准。
2. 它解决的是全场组织问题、教育目标问题、参与率问题、热血感问题，还是执行成本问题。
3. 它与“困境识别、行动生成、支持获得、导师认证”这条主线有什么关系。
4. 它会增加多少学习成本、物料成本和现场控制成本。
5. 它是否能从篮球馆空间、学生人数、导师关系和家长参与方式中自然推导出来。

全局设计按四层推进：

1. 目标层：活动要完成什么身份变化和教育结果。
2. 组织层：学生、导师、家长、领导分别处在什么位置，承担什么功能。
3. 空间层：篮球馆如何分区，学生如何移动，物料放在哪里。
4. 动作层：每个环节谁做什么，产出进入哪里，谁验收。

如果一个局部设计只能解释“可以解决某个小问题”，但无法解释它为什么属于整体结构，应先回到全局重新推导。

提出具体数字时，要说明数字来源。例如分几队、几条路线、几分钟、几张卡、几名代表，都要由人数、空间、时间、物料或安全边界推导出来。

每次方案迭代优先输出一张“全局结构表”，再展开流程细节。

## 新概念定义规则

提出任何新概念前，必须先判断它是否真的需要成为一个新词。

新概念包括但不限于：

- 新区域，例如任务岛、支持站、补给区。
- 新道具，例如装备卡、路线卡、行动条。
- 新角色，例如质检员、战报员、支援官。
- 新机制，例如闯关、解锁、认证、互审。

如果一个新词只是包装已有动作，应换成直接说法。例如“任务岛”应改成“小队任务区”，并说明它只是本队固定活动位置、物料摆放点和导师认证点。

确实需要保留的新概念，第一次出现时必须同时给出四项内容：

1. 一句话定义：它是什么。
2. 存在理由：它解决哪个全局问题。
3. 现场动作：谁在这里做什么。
4. 主线关系：它的结果进入“困境识别、行动生成、支持获得、导师认证”中的哪一步。

如果无法说清这四项内容，先不用这个新概念。

## 方案落地检查

任何新环节、道具、卡片、挑战、路线或评价机制进入方案前，必须回答六个问题：

1. 谁来做。
2. 做什么。
3. 为什么要做。
4. 怎么做。
5. 谁验收。
6. 结果进入哪里。

“结果进入哪里”必须指向明确产出，例如：

- 九年级行动牌的一栏。
- 小队任务板的一处。
- 家长支持卡。
- 导师认证记录。
- 现场荣誉评定。

## 三关设计约束

三关必须保持一条动作链：

1. 第一关产出“本队九年级困境”。
2. 第二关把该困境转化为两条可执行行动。
3. 第三关为这些行动获得同伴、家长、导师三方支持。
4. 结尾由导师认证整张行动牌。

每关只保留一个核心挑战、一个纸面产出、一个验收口令。上一关产出必须成为下一关输入。

## 评价规则

每个方案必须区分两层评价：

1. 入境达标：由本队导师判断，确保每队都能完成入境。
2. 荣誉评定：由运行教师或领导代表判断，用于区分完成质量。

入境达标至少包含：

- 选出一个具体九年级困境。
- 写出两条行动，每条行动包含“谁、何时、做什么、怎么检查”。
- 同伴、家长、导师三方支持齐全。
- 小队能用 30 秒说明自己的行动牌。

荣誉评定至少包含：

- 困境是否真实具体。
- 行动是否能立刻执行。
- 队内分工是否明显。
- 支持内容是否具体有效。
- 表达是否清楚。

## 写作要求

面向领导、老师、家长的方案要使用直接、朴素、可执行的语言。

优先写：

- 现场动作。
- 岗位分工。
- 时间节点。
- 道具数量。
- 验收标准。
- 突发处理。

谨慎使用抽象词、包装词和概念词。一个词如果需要额外解释才能听懂，应换成更直接的说法。
